## Presentation Content Rules

- **Slide content is short and punchy** — audience-facing, not speaker notes
- **Speaker notes are in HTML comments** — verbose, narrative, story-driven
- **No live demo** — examples and diagrams only
- **Never edit `output/index.html` directly** — it is generated by `build.cjs`

## Theming Guide

All theme customization is done in `build.cjs`. There are three places to change:

### 1. Fonts

Replace the `googleFonts` variable with your chosen Google Fonts link:

```js
const googleFonts = `
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link href="https://fonts.googleapis.com/css2?family=YOUR+FONT&display=swap" rel="stylesheet">
`;
```

Update the `font-family` references in `customCss` accordingly.

### 2. Color Variables

Edit the `:root` block inside `customCss`:

```css
:root {
  --ink: #18181b;       /* body text */
  --ink-dim: #52525b;   /* subtitles, secondary text */
  --muted: #a1a1aa;     /* captions, tertiary text */
  --line: #e4e4e7;      /* borders */
  --accent: #4f46e5;    /* primary highlights */
  --accent2: #059669;   /* secondary highlights */
  --radius: 28px;       /* slide corner radius */
}
```

### 3. Background

Edit the `html, body` rule inside `customCss`:

```css
html, body {
  background: radial-gradient(#f0ede8, #d8d2c8);
}
```

Keep it a single simple gradient — see the performance rules in `.agents/skills/markpress-styling/SKILL.md`.

### 4. Slide Card

Edit the `.step` base rule. Key constraints:
- `background` must be a **fully opaque hex** — no `rgba()`
- Do not add `box-shadow` to `.step` — only to `.step.active`
- Do not add pseudo-elements (`::before`, `::after`) to `.step`

## README Customization Guide

When customizing this repository for a specific talk, update `README.md` to include:

1. **Title and subtitle** — the talk name and one-sentence description
2. **Background section** — context for the talk, audience, and key ideas
3. **Render instructions** — keep the same build commands, update any prerequisites
4. **Talk outline** (optional) — a brief summary of each section
5. **Speaker bio** (optional) — a short bio and links

Remove or replace the "Use This Template" section once you've forked the template.

## Workflow

Always follow these steps when changing the presentation:

1. Plan the change
2. Make the change in `slides/presentation.md` or `build.cjs`
3. Run `npm run build` to render into HTML
4. Verify with visual QA — open `output/index.html` in the browser and check every slide
5. (Optional) If working in a cloud environment or commenting on a PR, include 3-5 screenshots
